# 安全分析标准参考文档

> 本文档定义 Web UI 安全侦察中阶段 6 的观察清单。
> 用于对照代码记录事实，不做风险评判。
> 可根据需要扩展观察维度。

---

## 维度 1：功能操作观察

识别前端中存在的功能操作。

### 观察清单

| # | 观察项 | 说明 |
|---|--------|------|
| 1.1 | 文件上传 | 是否存在 `<input type="file">` 或文件选择组件；上传的 API 端点；前端 accept 属性限制的文件类型 |
| 1.2 | 文件下载 | 下载 API 端点；响应处理方式 |
| 1.3 | 数据删除 | DELETE 请求；批量删除；是否有二次确认弹窗 |
| 1.4 | 权限变更 | 修改用户角色、权限的操作 |
| 1.5 | 资金/交易操作 | 涉及金额的操作（支付、转账、提现） |
| 1.6 | 用户状态变更 | 启用/禁用用户；锁定/解锁账户 |
| 1.7 | 配置修改 | 系统配置修改操作 |
| 1.8 | 二次认证（2FA） | 登录/高危操作是否存在二次认证环节。如存在，记录触发条件和类型（TOTP/SMS/生物识别/其他）；如不存在无需提及 |
| 1.9 | 二次编程/动态执行 | 应用是否允许动态代码执行，如：自定义公式/脚本、规则引擎、模板注入、宏/脚本上传、SQL 查询面板、代码在线编辑等 |
| 1.10 | 上传签名机制 | 文件上传是否要求签名/Token（如 STS 临时凭据、Presigned URL、上传凭证） |
| 1.11 | 上传文件格式 | 上传功能允许的文件类型，重点标注是否允许可执行文件/软件包：`.exe`, `.msi`, `.sh`, `.bat`, `.ps1`, `.apk`, `.dmg`, `.pkg`, `.bin`, `.jar`, `.war` 等 |

## 维度 2：输入校验规则

记录前端对用户输入的校验逻辑。

### 观察清单

| # | 观察项 | 说明 |
|---|--------|------|
| 2.1 | 类型校验 | `type="number"`、`type="email"` 等 HTML5 校验属性 |
| 2.2 | 长度/格式校验 | `minLength`、`maxLength`、`pattern` 属性或 JS 校验逻辑 |
| 2.3 | 校验位置 | 校验仅在前端实现，还是后端也校验 |
| 2.4 | 输入拼接 | 输入是否直接拼接到 URL 或请求体中 |
| 2.5 | XSS 相关 | `dangerouslySetInnerHTML`、`v-html` 等直接 HTML 输出的使用位置 |

---

## 维度 3：敏感信息输入

记录应用中涉及敏感信息输入的位置。

### 观察清单

| # | 观察项 | 说明 |
|---|--------|------|
| 3.1 | 密码输入 | `<input type="password">`；是否有明文显示选项 |
| 3.2 | 认证凭据 | API Key、Token、Access Key/Secret Key 等输入 |
| 3.3 | 个人身份信息 | 身份证号、手机号、邮箱地址、家庭住址等 PII 输入 |
| 3.4 | 支付信息 | 银行卡号、CVV、支付密码等 |
| 3.5 | 传输方式 | 凭据通过 HTTP 还是 HTTPS 传输；请求体中是否包含明文敏感信息 |

---

## 维度 4：URL/地址配置

记录前端中可配置 URL 地址的位置。

### 观察清单

| # | 观察项 | 说明 |
|---|--------|------|
| 4.1 | 用户可配置 URL | 系统配置中的站点 URL、回调 URL、Webhook URL 等 |
| 4.2 | 外部服务 Endpoint | 存储服务 Endpoint、外部 API 地址等 |
| 4.3 | 代理/转发配置 | 请求代理地址、API 网关地址等 |
| 4.4 | 图片/资源 URL | 用户头像 URL、封面图 URL 等（后端是否 fetch） |
| 4.5 | 导入/导出 URL | 数据导入导出的外部地址配置 |

---

## 维度 5：协议配置

记录应用中涉及的协议配置项。

### 观察清单

| # | 观察项 | 说明 |
|---|--------|------|
| 5.1 | 协议是否可配置 | 如 HTTP/HTTPS、TLS 版本、加密算法等 |
| 5.2 | 默认协议 | 默认使用的协议是什么 |
| 5.3 | 协议可选项 | 如 SMTP 可选 none/starttls/tls、WebSocket 可选 ws/wss |
| 5.4 | WebSocket 协议 | 使用 ws 还是 wss |

---

## 维度 6：错误处理

记录前端的错误处理机制和输入约束。

### 观察清单

| # | 观察项 | 说明 |
|---|--------|------|
| 6.1 | 校验规则 | 从 `rules`、`validators`、`yup/zod schema`、`pattern` 属性中提取输入约束条件 |
| 6.2 | 错误信息内容 | 记录具体错误文案，区分通用错误（"操作失败"）和具体错误（"用户名已被注册"） |
| 6.3 | 后端错误透传 | 后端错误信息是否直接展示给前端用户 |
| 6.4 | 认证错误 | 登录/认证时的错误信息是否区分"用户不存在"和"密码错误" |
| 6.5 | 反馈方式 | 错误通过什么方式反馈（弹窗、行内校验、toast、控制台、静默） |
| 6.6 | 错误中的敏感信息 | 错误信息/调试信息中是否包含：堆栈追踪、SQL 查询、文件路径、Token/凭据片段、API 密钥、内网地址等 |
